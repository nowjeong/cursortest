# 구구단 산성비 게임 PRD (Product Requirements Document)

## 1. 제품 개요

### 1.1 제품명
**구구단 산성비 게임** (Multiplication Rain)

### 1.2 목적
초등학교 2학년 학생들이 구구단을 재미있고 흥미롭게 학습할 수 있도록 돕는 교육용 게임

### 1.3 타겟 사용자
- 주 타겟: 초등학교 2학년 학생 (7-8세)
- 부 타겟: 구구단을 배우는 초등학교 저학년 학생

### 1.4 핵심 가치 제안
- 재미있는 게임을 통해 자연스럽게 구구단 학습
- 반복 학습을 통한 구구단 암기력 향상
- 성취감을 느끼며 학습 동기 부여

---

## 2. 게임 컨셉

### 2.1 게임 메커니즘
- 화면 위에서 구구단 문제가 적힌 빗방울이 떨어짐
- 플레이어는 화면 하단의 답 버튼을 클릭하여 정답을 맞춤
- 정답을 맞추면 빗방울이 사라지고 점수 획득
- 오답이거나 빗방울이 바닥에 닿으면 생명력 감소
- 모든 생명력을 잃으면 게임 오버

### 2.2 게임 목표
- 최대한 많은 구구단 문제를 맞춰 높은 점수 획득
- 단계별로 난이도를 높여가며 전체 구구단 마스터

---

## 3. 주요 기능 요구사항

### 3.1 메인 화면
**필수 기능:**
- 게임 시작 버튼
- 난이도 선택 (2단, 3단, ..., 9단, 전체 단)
- 최고 점수 표시
- 게임 방법 설명 버튼
- 귀여운 캐릭터 또는 일러스트

**UI 요구사항:**
- 밝고 친근한 색상 (파스텔톤 선호)
- 큰 버튼 (터치하기 쉽게)
- 명확한 아이콘과 텍스트

### 3.2 게임 플레이 화면

**필수 요소:**
```
┌─────────────────────────────────┐
│  점수: 0     ❤❤❤     시간: 60초  │
├─────────────────────────────────┤
│                                 │
│      [3×4]  ← 떨어지는 문제      │
│                                 │
│            [2×5]                │
│                                 │
│                                 │
├─────────────────────────────────┤
│  [6] [8] [10] [12] [15] [16]   │
│  [18] [20] [24] [27] [30] ...  │
└─────────────────────────────────┘
```

**게임 화면 구성:**
1. **상단 정보바**
   - 현재 점수
   - 남은 생명력 (하트 3개)
   - 남은 시간 또는 현재 레벨

2. **중앙 게임 영역**
   - 떨어지는 구구단 문제 (예: 3×4, 7×8)
   - 문제는 구름 또는 빗방울 모양 안에 표시
   - 떨어지는 속도는 난이도에 따라 조절
   - 동시에 2-4개의 문제가 떨어짐

3. **하단 답안 영역**
   - 가능한 답들을 버튼으로 표시
   - 선택한 단에 해당하는 답들 (예: 2단이면 2, 4, 6, 8, ..., 18)
   - 정답과 오답이 섞여 있음
   - 버튼 크기: 손가락으로 누르기 쉬운 크기

### 3.3 난이도 설정

**레벨 구성:**
| 레벨 | 구구단 범위 | 떨어지는 속도 | 동시 문제 수 | 시간 제한 |
|------|------------|--------------|-------------|----------|
| 입문 | 2단, 5단 | 느림 | 1-2개 | 90초 |
| 초급 | 2-5단 | 느림 | 2-3개 | 90초 |
| 중급 | 2-9단 (쉬운 것) | 보통 | 2-3개 | 60초 |
| 고급 | 2-9단 (전체) | 빠름 | 3-4개 | 60초 |
| 달인 | 2-9단 (전체) | 매우 빠름 | 4-5개 | 45초 |

### 3.4 점수 시스템

**점수 계산:**
- 정답: +10점
- 콤보 보너스: 연속 정답 시 추가 점수
  - 5연속: +50점 보너스
  - 10연속: +100점 보너스
- 빠른 정답: 문제가 떨어진 직후 맞추면 +5점 추가
- 오답: 점수 차감 없음 (생명력만 감소)

**생명력 시스템:**
- 시작 생명력: 3개 (하트 3개)
- 생명력 감소 조건:
  - 오답 선택: -1
  - 문제가 바닥에 닿음: -1
- 생명력이 0이 되면 게임 오버

### 3.5 게임 종료 화면

**표시 정보:**
- 최종 점수
- 최고 점수 (갱신 시 애니메이션 효과)
- 맞춘 문제 수 / 전체 문제 수
- 정확도 (%)
- 격려 메시지 (점수에 따라 다름)
  - 0-50점: "다시 도전해보자! 💪"
  - 51-100점: "잘했어요! 조금만 더! 🌟"
  - 101-200점: "훌륭해요! 구구단 박사! 🎉"
  - 201점 이상: "완벽해요! 구구단 천재! 🏆"

**버튼:**
- 다시 하기
- 난이도 변경
- 메인으로 돌아가기

### 3.6 사운드 및 효과

**사운드:**
- 배경 음악: 경쾌하고 밝은 음악
- 정답 효과음: 긍정적인 "띠링!" 소리
- 오답 효과음: 부드러운 "앗!" 소리 (너무 부정적이지 않게)
- 콤보 달성: 특별한 "짜잔!" 소리
- 게임 오버: 부드러운 종료 음

**시각 효과:**
- 정답 시: 반짝이는 별 애니메이션
- 오답 시: 부드러운 흔들림 효과
- 콤보 달성: 화면 전체 반짝임
- 레벨업: 불꽃놀이 효과

## 4. 기술 요구사항

### 4.1 플랫폼
**우선순위 1:**
- 웹 브라우저 (크롬, 사파리, 엣지 등)
- 반응형 디자인 (PC, 태블릿, 모바일)

**우선순위 2:**
- 안드로이드 앱
- iOS 앱

### 4.2 기술 스택 (예시)
- **프론트엔드:** HTML5, CSS3, JavaScript (또는 React)
- **게임 엔진:** Phaser.js 또는 PixiJS (선택사항)
- **애니메이션:** CSS Animation 또는 GSAP
- **로컬 저장소:** LocalStorage (최고 점수 저장)

### 4.3 성능 요구사항
- 로딩 시간: 3초 이내
- 프레임률: 60 FPS 유지
- 터치/클릭 반응 속도: 50ms 이내

## 5. UI/UX 디자인 가이드라인

### 5.1 디자인 원칙
1. **직관성:** 아이들이 설명 없이도 이해할 수 있어야 함
2. **피드백:** 모든 행동에 즉각적인 시각/청각 피드백 제공
3. **안전성:** 실패해도 부담 없이 다시 도전할 수 있는 분위기
4. **재미:** 게임적 요소를 통한 학습 동기 부여

### 5.2 색상 팔레트
- **주 색상:** 하늘색 (#87CEEB), 파스텔 블루 (#B0E0E6)
- **강조 색상:** 노란색 (#FFD700), 초록색 (#90EE90)
- **배경:** 밝은 흰색 (#FFFFFF) 또는 연한 하늘색
- **텍스트:** 짙은 남색 (#191970) - 가독성 확보

### 5.3 타이포그래피
- **글꼴:** 둥글고 친근한 산세리프체 (예: Noto Sans KR, 배달의민족 도현체)
- **문제 텍스트:** 40-50px (크고 명확하게)
- **답안 버튼:** 30-35px
- **점수/정보:** 20-25px

### 5.4 애니메이션
- 부드러운 움직임 (ease-in-out)
- 과하지 않은 효과 (집중력 분산 방지)
- 문제 떨어지는 속도는 예측 가능하게

## 6. 추가 기능 (선택사항)

### 6.1 학습 모드
- **연습 모드:** 시간 제한 없이 특정 단 연습
- **오답 노트:** 틀린 문제를 모아서 다시 풀기
- **진도 추적:** 각 단별 정확도와 진행도 표시

### 6.2 보상 시스템
- **별 수집:** 일정 점수마다 별 획득
- **배지 시스템:** 
  - "2단 마스터", "5연속 정답", "완벽한 게임" 등
- **캐릭터 꾸미기:** 모은 별로 캐릭터 꾸미기 아이템 구매

### 6.3 멀티플레이어
- 친구와 점수 경쟁
- 온라인 리더보드

### 6.4 학부모 대시보드
- 자녀의 학습 진행도 확인
- 취약한 구구단 파악
- 학습 시간 통계

## 7. 개발 우선순위

### Phase 1 (MVP - 최소 기능 제품)
1. 기본 게임 플레이 (단일 난이도)
2. 점수 시스템
3. 생명력 시스템
4. 2-5단 구구단
5. 기본 UI/UX

### Phase 2 (핵심 기능 추가)
1. 난이도 선택 (5단계)
2. 전체 구구단 (2-9단)
3. 사운드 효과
4. 최고 점수 저장
5. 게임 방법 튜토리얼

### Phase 3 (향상된 경험)
1. 콤보 시스템
2. 다양한 시각 효과
3. 배경 음악
4. 연습 모드
5. 모바일 최적화

### Phase 4 (추가 가치)
1. 보상 시스템
2. 오답 노트
3. 진도 추적
4. 다양한 테마
5. 학부모 대시보드

## 8. 성공 지표 (KPI)

### 8.1 사용자 참여도
- 일일 활성 사용자 (DAU)
- 평균 게임 플레이 시간: 10분 이상
- 재방문율: 주 3회 이상
- 게임당 평균 플레이 횟수: 3회 이상

### 8.2 학습 효과
- 난이도 상승률 (입문 → 고급)
- 평균 정확도 향상률
- 전체 구구단 달성률

### 8.3 사용자 만족도
- 앱 스토어 평점: 4.5점 이상
- 학부모 피드백 긍정률: 80% 이상
- 완료율 (게임을 끝까지 플레이): 70% 이상

## 9. 리스크 및 고려사항

### 9.1 학습 효과
- **리스크:** 게임만 하고 학습 효과가 적을 수 있음
- **대응:** 연습 모드와 오답 노트로 복습 기회 제공

### 9.2 난이도 밸런싱
- **리스크:** 너무 쉽거나 어려워서 흥미 상실
- **대응:** 사용자 테스트를 통한 난이도 조절, 자동 난이도 조절 기능

### 9.3 집중력 유지
- **리스크:** 반복적인 게임으로 지루함
- **대응:** 다양한 보상, 새로운 테마, 일일 챌린지 제공

### 9.4 접근성
- **리스크:** 학습 장애가 있는 아동의 접근성
- **대응:** 음성 지원, 속도 조절 옵션, 색맹 모드
