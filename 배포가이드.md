## ⚡ 빠른 배포 (3단계)

### 1단계: Git 설치 확인 및 설치

#### Git 설치 확인
터미널(PowerShell)을 열고 실행:
```bash
git --version
```

**오류가 나온다면** Git을 설치해야 합니다.

#### Git 설치 방법

**방법 A: 공식 사이트에서 다운로드 (추천)**
1. https://git-scm.com/download/win 접속
2. "64-bit Git for Windows Setup" 다운로드
3. 설치 프로그램 실행 (모든 옵션 기본값으로 설치)
4. **설치 후 반드시 터미널 재시작!**

**방법 B: winget 사용 (Windows 10/11)**
```bash
winget install Git.Git
```

### 2단계: Git 설정 및 코드 업로드

**새 PowerShell 터미널을 열고** 프로젝트 폴더로 이동한 후:

```bash
# Git 사용자 정보 설정
git config --global user.email "zerg1025@dge.go.kr"
git config --global user.name "nowjeong"

# Git 저장소 초기화
git init

# 모든 파일 추가
git add .

# 첫 커밋
git commit -m "Initial commit: 구구단 산성비 게임 완성"

# 기본 브랜치를 main으로 설정
git branch -M main

# GitHub 저장소 연결
git remote add origin https://github.com/nowjeong/cursortest.git

# GitHub에 푸시
git push -u origin main
```

**중요**: `git push` 실행 시 GitHub 로그인 창이 나타나면 로그인하세요!

### 3단계: GitHub Pages 활성화

1. 브라우저에서 https://github.com/nowjeong/cursortest 접속
2. 상단 메뉴에서 **"Settings"** 클릭
3. 왼쪽 메뉴에서 **"Pages"** 클릭
4. **"Source"** 드롭다운에서 **"GitHub Actions"** 선택
5. 자동으로 배포 시작!

### 4단계: 배포 완료 확인

1. 저장소 상단의 **"Actions"** 탭 클릭
2. 워크플로우가 실행되는지 확인 (초록색 체크 표시 기다리기)
3. 약 2-3분 후 완료!
4. **게임 접속**: https://nowjeong.github.io/cursortest/

## 🛠️ 문제 해결

### ❌ 문제 1: "git: command not found" 오류

**해결책**: Git이 설치되지 않았습니다.
- 위의 "Git 설치 방법" 참고
- 설치 후 **반드시 터미널 재시작**

### ❌ 문제 2: GitHub 로그인 요청

**해결책**: 
1. GitHub 계정으로 로그인
2. 또는 Personal Access Token 생성:
   - GitHub → Settings → Developer settings → Personal access tokens
   - "Generate new token (classic)"
   - "repo" 권한 체크
   - 생성된 토큰을 비밀번호 대신 입력

### ❌ 문제 3: 배포 후 페이지가 비어있음

**해결책**: `vite.config.ts`의 `base` 경로 확인
- 현재 설정: `base: '/cursortest/'`
- 저장소 이름과 일치해야 함!

### ❌ 문제 4: Actions 탭에서 배포 실패

**해결책**:
1. Actions 탭에서 에러 로그 확인
2. Settings → Pages에서 Source가 "GitHub Actions"인지 확인
3. package.json에 오류가 있는지 확인
